# ddd-vsa-hex-typescript-tauri template

A DDD-VSA-Hex TypeScript + Rust (Tauri 2) scaffold aligned with
`docs/design.md` §6 / §12 / §17. Derives from `ddd-vsa-hex-typescript` and
adds a Rust backend under `apps/<app>/src-tauri/` plus a `tauri-specta`
cross-root contract that auto-generates TS bindings.

The TS<->Rust language boundary IS the cross-slice isolation boundary —
enforced physically by the compiler, not by lint. Each root in
`.ori/architecture.md` picks its own adapter (eslint for TS,
arch-adapter-rust for Rust).

## Layout

```
<project>/
├── .ori/architecture.md                  # SSoT — two roots: ts + rs
├── eslint.config.js                      # spreads ./eslint.config.ori.js (ts root)
├── vitest.config.ts
├── tsconfig.json
└── apps/
    └── template-app/                     # default app (rename per /ori-init)
        ├── src/                          # TS frontend
        │   ├── task-management/
        │   │   ├── shared/
        │   │   │   ├── ipc/              # tauri-specta-generated bindings
        │   │   │   ├── types/
        │   │   │   ├── events/
        │   │   │   └── contracts/
        │   │   └── slices/
        │   │       └── complete-task/    # one worked slice (TS side)
        │   ├── ui-widget/                # ddd-vsa-hex ui-layer (order 1)
        │   └── ui-page/                  # ddd-vsa-hex ui-layer (order 2)
        └── src-tauri/                    # Rust backend
            ├── Cargo.toml                # generated by 'tauri init' (delegated)
            ├── tauri.conf.json           # generated by 'tauri init' (delegated)
            ├── build.rs                  # generated by 'tauri init' (delegated)
            ├── capabilities/             # generated by 'tauri init' (delegated)
            └── src/
                ├── lib.rs                # ori overlay (tauri-specta wiring) — kept across tauri init
                ├── main.rs               # generated by 'tauri init' (delegated)
                └── task_management/      # Rust BC mirror — note underscore (Rust ident rules)
                    ├── shared/
                    └── slices/
                        └── complete_task/  # mirrors the TS slice
```

`ori init --template ddd-vsa-hex-typescript-tauri` copies only the **ori
overlay** (everything above marked `ori overlay`) and then shells out to
`pnpm tauri init --ci ...` so Tauri owns its own version-volatile files
(`Cargo.toml`, `tauri.conf.json`, `build.rs`, `capabilities/`, `src/main.rs`).
This means a `tauri-cli` bump never forces an ori template re-release.

## Getting started

`ori init` runs `pnpm install` and `pnpm tauri init` for you. If you passed
`--skip-tauri-init`, finish the bootstrap manually:

```bash
pnpm install
pnpm tauri init --ci \
  --app-name my-app --window-title my-app \
  --frontend-dist ../dist --dev-url http://localhost:5173 \
  --before-dev-command "pnpm vite" --before-build-command "pnpm vite build" \
  --directory apps/template-app/src-tauri
```

Then:

```bash
pnpm arch:export                 # regenerate eslint.config.ori.js + tests/arch.rs
pnpm test                        # TS tests (vitest)
pnpm test:rs                     # Rust tests (cargo)
pnpm tauri dev                   # also regenerates task-management/shared/ipc/bindings.ts
```

## Conventions

### TypeScript frontend (same as ddd-vsa-hex-typescript)

- **Value Objects** are branded types with a Smart Constructor returning
  `Result<T, Error>`.
- **Aggregates** are pure: command functions take state, return
  `{ state, events }`.
- **Public API**: a slice's `index.ts` is the only file that may be
  imported from outside the slice.
- **UI layers** flow one-way: `ui-page -> ui-widget -> {shared, domain}`.
  UI layers reach a slice through the slice's public `index.ts`; the
  tauri-specta bindings exposed under `task-management/shared/ipc/` are
  the legal cross-root entry point.

### Rust backend

- Each slice folder owns its own pipeline: `domain.rs -> application.rs ->
  infrastructure.rs -> commands.rs`.
- **`mod.rs` is the public API** for the slice. The crate root re-exports
  the BC module (`task_management`); anything outside `mod.rs` is
  slice-internal.
- **Tauri commands** (`#[tauri::command] #[specta::specta]`) live in
  `commands.rs`. `tauri-specta` collects them at build time and emits
  `apps/<app>/src/task-management/shared/ipc/bindings.ts` (the only
  cross-root artifact).
- Use idiomatic `super::*` for sibling modules inside a slice; reach for
  `crate::task_management::shared::*` only when you need a BC-internal
  shared primitive.

### Cross-slice collaboration

- **Direct imports across slices are prohibited** on both sides. Declare
  a contract in `task-management/shared/contracts/` (TS) or
  `task_management::shared` (Rust), or publish a domain event, and have
  both sides depend on the contract.

## Customising

Edit `.ori/architecture.md` to add layers, slices, or change rules; then:

```bash
pnpm arch:export
```

The generated files (`eslint.config.ori.js`,
`apps/template-app/src-tauri/tests/arch.rs`) are in `.gitignore` by
default — the spec is the source of truth.
